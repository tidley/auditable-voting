## Responses view - literal UI spec

This spec is strict.

The **Responses** tab contains only the items below, in this order.

No extra sections.
No second-level navigation.
No editing controls.

---

# Page title

## Heading

**Responses**

Optional subtext:
**View submitted responses and publish the response summary.**

---

# Page structure

## Section 1 - Questionnaire selector

### Heading

**Questionnaire**

### Row order

1. **Questionnaire dropdown**

### Component block

* Type: dropdown / select
* Label: `Questionnaire`
* Behaviour:

  * lists available questionnaires
  * selecting one refreshes all data below
  * if only one exists, still show it in the same position

### Example layout

* Label above
* Full-width dropdown below

---

## Section 2 - Metadata

### Heading

**Metadata**

### Row order

1. Questionnaire ID
2. State
3. Opened
4. Closing / Closed

### Component block

Use a compact metadata grid with four rows or two columns.

#### Fields

* **Questionnaire ID**

  * read-only text
  * copy button optional
* **State**

  * Draft / Published / Open / Closed / Counted
* **Opened**

  * date/time or `Not opened`
* **Closing / Closed**

  * date/time or `Not scheduled`

### Layout

Preferred:

* two-column metadata grid on desktop
* one-column stacked layout on mobile/narrow view

---

## Section 3 - Response summary

### Heading

**Responses**

### Row order

1. response count
2. completion percentage

### Component block

Single summary strip or card.

#### Fields

* **Responses**

  * format: `n / m`
* **Completion**

  * format: `x%`

Optional:

* simple progress bar under the values

### Layout

* one horizontal summary card
* large numeric emphasis
* not interactive

---

## Section 4 - Publish action

### Heading

No separate heading needed if visually obvious, otherwise:

**Publish**

### Row order

1. Publish button
2. publish status text

### Component block

#### Primary button

**Publish summary**

#### Helper/status text below button

Possible states:

* `Ready to publish`
* `Already published`
* `Nothing to publish yet`
* `Publishing...`
* `Publish failed`

### Behaviour

* hashes and submits the response summary/publication artefact
* disabled if publishing is not valid
* visible result after click

### Layout

* button aligned left
* status text directly underneath or to the right

---

## Section 5 - Question results

### Heading

**Question results**

### Row order

Questions appear in questionnaire order:

1. Question 1 result block
2. Question 2 result block
3. Question 3 result block
4. etc.

### Component block for each question

Each question must be a distinct card.

#### Card header

* **Question X**
* question type badge:

  * `Yes / No`
  * `Multiple choice`
  * `Text`

#### Card body

* **Prompt**
* result visualisation

---

### Result visualisation rules

## Yes / No question block

Show:

* prompt
* bar chart or split row
* percentages and counts

#### Exact contents

1. Prompt text
2. Horizontal result bar
3. Result labels:

   * `Yes - 70% (28)`
   * `No - 30% (12)`

---

## Multiple choice question block

Show:

* prompt
* one row per option
* bar and percentage per option

#### Exact contents

1. Prompt text
2. Option row 1
3. Option row 2
4. etc.

#### Each option row contains

* option label
* horizontal bar
* percentage
* count

Example:

* `Too slow - 25% (10)`
* `About right - 60% (24)`
* `Too fast - 15% (6)`

---

## Text question block

Show:

* prompt
* response count
* expandable list of text responses

#### Exact contents

1. Prompt text
2. `Responses: n`
3. `Show responses` expand/collapse control
4. expanded text response list if opened

Do not show all text responses by default if there are many.

---

## Question card spacing

Required:

* generous vertical spacing between cards
* clear border or background contrast
* prompt clearly separated from results

---

## Section 6 - Responders

### Heading

**Responders**

### Purpose

Show who has actually responded.

### Inclusion rule

Only include responders who have submitted a valid response.

Do not show all invited voters here by default.

### Sorting

Alphabetical by responder ID.

### Row order

1. responder row A
2. responder row B
3. responder row C
4. etc.

### Component block for each responder row

Each row contains:

1. **coloured QR-style visual marker**
2. **Responder ID**
3. optional submission timestamp or status

### Exact row structure

* left: coloured QR-style marker / square token
* middle: responder ID text
* right: optional secondary metadata such as:

  * `Submitted`
  * submission time

### Example row

* `[QR marker]  b93cb43   Submitted 14:22`

### Empty state

If no responses yet:
**No responders yet. Responders will appear here after submitting a response.**

---

# Full page order summary

Use this exact order:

1. **Responses** page heading
2. **Questionnaire** section

   * questionnaire dropdown
3. **Metadata** section

   * Questionnaire ID
   * State
   * Opened
   * Closing / Closed
4. **Responses** summary section

   * `n / m`
   * `x%`
   * optional progress bar
5. **Publish** action

   * `Publish summary` button
   * status text
6. **Question results**

   * one card per question in order
7. **Responders**

   * alphabetical responder list with coloured QR-style markers

---

# Explicit exclusions

Do not include on the Responses page:

* add question buttons
* question editing controls
* reorder / duplicate / delete question actions
* coordinator management
* invite voters
* QR invite tools
* protocol debug details
* settings controls
* second-row submenu

---

# Behaviour summary

## On tab open

* load selected questionnaire
* render metadata
* render response summary
* render question result cards
* render responder list alphabetically

## On publish click

* disable button while publishing
* show status update
* show success/failure result

## On responder updates

* append newly valid responders
* keep list alphabetically sorted
* preserve coloured QR-style identity marker
