This question UI is functional, but it feels like raw form controls rather than a polished questionnaire.

## Main problems

* The three questions run together too much.
* Prompts like **Have**, **which**, **writeit** are visually weak and look like labels rather than actual questions.
* The yes/no buttons are oversized relative to the rest and dominate the whole page.
* Multiple choice options are too small and low-emphasis.
* The free-text box feels detached from the question and too plain.
* There is no clear card or boundary per question.

## Better structure

Each question should be its own **question card** with:

* question number
* full prompt
* optional helper text
* answer control
* clear spacing below before the next question

For example:

### Question 1

**Have you enjoyed the course?**
[ Yes ] [ No ]

### Question 2

**Which parts helped most?**
[ ] Option 1
[ ] Option 2
[ ] Option 3

### Question 3

**Write any additional comments**
[ text area ]

That immediately makes the page easier to scan.

## Specific improvements

### 1. Make prompts much more prominent

Use:

* larger font
* bolder weight
* more spacing below prompt

The prompt should be the first thing the eye lands on.

### 2. Put each question in a separate card

Each card should have:

* subtle border or darker panel
* padding around contents
* generous gap between cards

This will stop the questions blending together.

### 3. Reduce the visual aggression of the yes/no buttons

The yes/no buttons are too dominant.

Improve by:

* reducing their height slightly
* reducing horizontal width so they do not stretch edge-to-edge
* keeping strong selected state, but making the unselected state calmer

They should feel like answer choices, not page banners.

### 4. Improve multiple choice readability

The checkboxes and labels are too small and too close together.

Improve by:

* larger click targets
* more vertical spacing between options
* putting each option on its own row with stronger alignment
* optionally wrapping each option in a subtle selectable row

### 5. Improve the free-text area

The text box should be more obviously tied to the question.

Improve by:

* adding spacing above it
* adding placeholder text like:

  * `Type your response here...`
* making the label clearer, for example:

  * **Additional comments**
* placing max-character text in a lighter secondary style beneath the box

### 6. Add question numbering

Show:

* `Question 1`
* `Question 2`
* `Question 3`

This helps orientation, especially for longer questionnaires.

### 7. Add required/optional indicators where relevant

For example:

* `Required`
* `Optional`

That helps users understand whether they can skip a field.

## Recommended visual hierarchy

For each question card:

1. **Question number**
2. **Prompt**
3. helper/secondary text if needed
4. answer control
5. validation/help text

## Best practical redesign

Use this pattern strictly:

* one question per card
* prompt bold and prominent
* answer area directly below
* large vertical spacing between cards
* consistent control sizing across question types

So instead of the current compressed flow:

* prompt
* giant buttons
* prompt
* tiny checkboxes
* prompt
* text box

make it feel like a structured questionnaire with clear blocks.

## Short version

Improve it by:

* separating each question into a card
* making prompts much larger and clearer
* toning down the yes/no buttons
* making checkboxes easier to read and click
* visually linking the text area to its question
* adding numbering and spacing throughout
